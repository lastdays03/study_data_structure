## 📚 자료구조 개념 및 구현 (4주)

**대상**: 기본 프로그래밍 능력이 있는 중급자  
**교재**: 자료구조와 함께 배우는 알고리즘 입문  
**기간**: 4주 (1개월)  
**목표**: 자료구조의 근본 개념 이해 및 직접 구현 능력 확보

### 주차별 상세 계획

**1주차: 기본 자료구조 - 배열, 리스트, 스택, 큐, 덱**
- **학습 목표**: 기본 자료구조의 동작 원리와 구현 방법 이해
- **핵심 내용**:
  - 배열의 메모리 구조와 접근 방식
  - 동적 배열의 동작 원리 및 확장 메커니즘
  - 연결 리스트의 구조 (단일, 이중, 원형)
  - 스택의 LIFO 구조와 활용
  - 큐의 FIFO 구조와 활용
  - 덱(Deque)의 구조와 활용
  - 각 자료구조의 시간/공간 복잡도 분석
- **실습**: 
  - 배열, 연결 리스트, 스택, 큐, 덱 직접 구현
  - 각 자료구조의 기본 연산 구현 (삽입, 삭제, 탐색)
  - 테스트 코드 작성 및 검증
- **체크리스트**:
  - [ ] 배열과 연결 리스트의 차이점 및 장단점 이해
  - [ ] 연결 리스트 직접 구현 가능 (단일, 이중)
  - [ ] 스택과 큐 직접 구현 가능
  - [ ] 각 자료구조의 시간 복잡도 설명 가능
  - [ ] 각 자료구조의 활용 사례 이해

**2주차: 트리 구조 - 이진 트리, BST, 힙**
- **학습 목표**: 트리 구조의 동작 원리와 구현 방법 이해
- **핵심 내용**:
  - 트리의 기본 개념과 용어 (노드, 엣지, 루트, 리프 등)
  - 이진 트리의 구조와 특성
  - 트리 순회 방법 (전위, 중위, 후위, 레벨 순회)
  - 이진 탐색 트리 (BST)의 동작 원리
  - BST의 삽입, 삭제, 탐색 연산
  - 힙(Heap)의 구조와 동작 원리
  - 최소 힙과 최대 힙
  - 힙 정렬 알고리즘
- **실습**: 
  - 이진 트리 직접 구현
  - BST 직접 구현 (삽입, 삭제, 탐색)
  - 힙 직접 구현 (최소 힙, 최대 힙)
  - 트리 순회 알고리즘 구현
  - 힙 정렬 구현
- **체크리스트**:
  - [ ] 트리의 기본 구조와 용어 이해
  - [ ] 트리 순회 알고리즘 구현 가능
  - [ ] BST 직접 구현 가능 (삽입, 삭제, 탐색)
  - [ ] 힙 직접 구현 가능
  - [ ] 각 자료구조의 시간 복잡도 분석 가능

**3주차: 그래프 구조 - 인접 리스트/행렬, DFS/BFS, 최단 경로**
- **학습 목표**: 그래프의 표현 방법과 기본 알고리즘 이해
- **핵심 내용**:
  - 그래프의 기본 개념 (정점, 간선, 방향/무방향 그래프)
  - 인접 리스트 vs 인접 행렬 비교
  - 각 표현 방법의 장단점 및 시간/공간 복잡도
  - 그래프 순회 (DFS, BFS)
  - DFS/BFS의 구현 방법 및 활용
  - 최단 경로 알고리즘 기초 (다익스트라, 플로이드-워셜)
  - 위상 정렬 (Topological Sort)
- **실습**: 
  - 그래프 인접 리스트/행렬 구현
  - DFS/BFS 알고리즘 구현
  - 다익스트라 알고리즘 구현
  - 플로이드-워셜 알고리즘 구현
  - 위상 정렬 구현
- **체크리스트**:
  - [ ] 그래프의 기본 개념 이해
  - [ ] 인접 리스트와 인접 행렬 구현 가능
  - [ ] DFS/BFS 구현 및 활용 가능
  - [ ] 최단 경로 알고리즘 구현 가능
  - [ ] 각 알고리즘의 시간 복잡도 분석 가능

**4주차: 해시 테이블, 정렬 알고리즘, 고급 자료구조**
- **학습 목표**: 해시 테이블, 정렬 알고리즘, 고급 자료구조 이해
- **핵심 내용**:
  - 해시 함수의 동작 원리
  - 해시 충돌 해결 방법 (체이닝, 개방 주소법)
  - 주요 정렬 알고리즘 (버블, 선택, 삽입, 병합, 퀵, 힙)
  - 각 정렬 알고리즘의 시간/공간 복잡도 비교
  - 안정 정렬 vs 불안정 정렬
  - 고급 자료구조 소개 (트라이, 세그먼트 트리, 펜윅 트리)
- **실습**: 
  - 해시 테이블 직접 구현 (체이닝 방식)
  - 주요 정렬 알고리즘 직접 구현
  - 정렬 알고리즘 성능 비교 및 벤치마킹
  - 트라이(Trie) 기본 구현
- **체크리스트**:
  - [ ] 해시 테이블의 동작 원리 이해
  - [ ] 해시 테이블 직접 구현 가능
  - [ ] 주요 정렬 알고리즘 구현 가능
  - [ ] 각 알고리즘의 시간 복잡도 설명 가능
  - [ ] 정렬 알고리즘 성능 비교 가능

**최종 프로젝트**: 
- 자료구조 라이브러리 직접 구현 (배열, 리스트, 스택, 큐, 트리, 그래프, 해시 테이블)
- 각 자료구조의 시간/공간 복잡도 분석 문서 작성
- 테스트 코드 및 벤치마크 작성

- **복잡도 분석**: 모든 자료구조의 시간/공간 복잡도 상세 분석

### 2. 구현 우선 원칙
- **이론 30% + 구현 60% + 복습 10%**: 이론 학습 후 즉시 구현

- **테스트 코드 작성**: 구현한 자료구조에 대한 테스트 코드 작성
- **에러 해결 경험**: 에러 발생 시 해결 과정 기록 및 분석

### 3. 복습 및 정리
- **주말 복습**: 주간 학습 내용 정리 및 약점 보완
- **구현 코드 정리**: 구현한 자료구조 코드를 주제별로 정리
- **복잡도 분석 문서화**: 각 자료구조의 복잡도 분석 문서 작성
- **스터디 그룹**: 다른 학습자와 구현 코드 공유 및 리뷰 (선택)

### 4. 심화 학습
- **최적화 연습**: 구현한 자료구조의 성능 최적화
- **벤치마킹**: 다양한 자료구조의 성능 비교 및 벤치마킹
- **실전 활용**: 구현한 자료구조를 활용한 간단한 애플리케이션 개발

## 마무리

이 학습 플랜은 **중급자**를 대상으로 자료구조의 **개념과 구현**을 **1개월 이내**에 체계적으로 학습할 수 있도록 최적화되었습니다.

**학습 개요:**
- **대상**: 기본 프로그래밍 능력이 있는 중급자
- **교재**: 자료구조와 함께 배우는 알고리즘 입문 (1권)
- **기간**: 4주 (1개월)
- **목표**: 자료구조 개념과 기본 구현 중심

꾸준한 구현 연습과 복습을 통해 자료구조 역량을 키워나가시기 바랍니다.

**성공의 핵심:**
1. 매일 조금씩이라도 구현 연습 (습관화)
2. 이론과 구현의 균형 (이론 30% + 구현 60%)
3. 직접 구현 중심 학습 (라이브러리 사용 금지)
4. 주말 복습으로 개념 정착
5. 테스트 코드 작성으로 검증
6. 복잡도 분석 문서화로 이해도 향상
