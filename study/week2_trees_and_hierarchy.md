# 2주차: 트리 및 계층적 자료구조

## 📌 학습 목표
- 계층적 데이터를 표현하는 트리(Tree) 구조를 이해합니다.
- 재귀(Recursion)를 활용하여 트리를 순회(Traversal)하는 방법을 익힙니다.
- 이진 탐색 트리(BST)와 힙(Heap)의 동작 원리를 이해하고 구현합니다.

## 📚 핵심 학습 내용 (Ch 07 ~ Ch 08)

### 1. 트리 (Tree) 기초 (Ch 07)
- **개념**: 노드(Node)들이 계층적으로 연결된 비선형 자료구조입니다.
- **용어**:
  - **Root**: 최상위 노드
  - **Parent/Child**: 부모/자식 관계
  - **Leaf**: 자식이 없는 말단 노드
  - **Depth/Height**: 깊이와 높이
- **표현 방법**:
  - **배열 표현법**: 완전 이진 트리에 적합 (인덱스 규칙 활용)
  - **링크 표현법**: 노드와 참조(링크)를 이용, 일반적인 트리 구현에 사용

### 2. 트리 순회 (Tree Traversal)
- 트리의 모든 노드를 한 번씩 방문하는 방법입니다.
- **재귀(Recursion)**를 이용하면 코드가 간결해집니다.
- **종류**:
  - **전위 순회 (Pre-order)**: Root -> Left -> Right
  - **중위 순회 (In-order)**: Left -> Root -> Right
  - **후위 순회 (Post-order)**: Left -> Right -> Root

### 3. 이진 탐색 트리 (BST, Binary Search Tree)
- **특징**:
  - 왼쪽 자식 노드 < 부모 노드 < 오른쪽 자식 노드
  - 중위 순회 시 정렬된 값을 얻을 수 있습니다.
- **연산**:
  - **탐색**: 이진 탐색과 유사하게 O(log n) 성능 (균형 잡힌 경우)
  - **삽입**: 탐색 실패 위치에 새 노드 추가
  - **삭제**: 자식 노드의 개수(0, 1, 2)에 따라 처리 로직이 다름

### 4. 힙 (Heap)
- **개념**: 완전 이진 트리의 일종으로, 우선순위 큐를 구현하는 데 사용됩니다.
- **종류**:
  - **최대 힙 (Max Heap)**: 부모 >= 자식
  - **최소 힙 (Min Heap)**: 부모 <= 자식
- **활용**: 힙 정렬, 최단 경로 알고리즘(다익스트라) 등
